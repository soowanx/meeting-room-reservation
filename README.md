# 회의실 예약 관리 웹 어플리케이션

사용자는 회의실을 조회하고 예약할 수 있고, 관리자는 회의실과 전체 예약을 관리할 수 있는 Spring Boot 기반 웹 애플리케이션입니다.

## 주요 기능

- 회의실 목록 조회
- 회의실별 예약 생성
- 중복 시간대 예약 방지
- 이메일 기반 내 예약 조회 및 취소
- 관리자 로그인
- 회의실 등록 및 활성화/비활성화
- 전체 예약 조회 및 관리자 취소

## 기술 스택

- Backend: Java 17, Spring Boot, Spring MVC, Spring Data JPA, Thymeleaf
- Database: PostgreSQL
- Infra: Docker, Docker Compose

## 실행 방법

프로젝트 루트에서 아래 명령어를 실행합니다.

```bash
docker compose up --build
```

실행 후 접속 주소:

- 사용자 화면: http://localhost:8080
- 관리자 로그인: http://localhost:8080/admin/login

기본 관리자 비밀번호:

```text
admin1234
```

원하면 `docker-compose.yml`의 `APP_ADMIN_PASSWORD` 값을 변경해서 사용할 수 있습니다.

## 설계 내용

### 1. 사용자와 관리자 동선을 분리

- 사용자는 로그인 없이 이름과 이메일만으로 빠르게 예약할 수 있게 했습니다.
- 관리자는 별도 로그인 화면을 통해 회의실과 전체 예약을 관리하도록 분리했습니다.

과제 시간 안에 핵심 사용 흐름을 완성하는 것이 우선이었기 때문에, 일반 사용자 인증보다는 예약 생성과 조회 경험을 더 간단하게 두었습니다.

### 2. 예약 중복 방지 로직

가장 중요한 핵심 요구사항을 회의실 중복 예약 방지로 보고 아래 조건으로 처리했습니다.

```text
새 예약 시작 < 기존 예약 종료 && 새 예약 종료 > 기존 예약 시작
```

예약 생성 시 같은 회의실의 확정 예약 중 시간이 겹치는 데이터가 있는지 조회해서 차단합니다.

### 3. 엔티티 구성

#### Room

- 이름
- 위치
- 수용 인원
- 설명
- 활성화 여부

#### Reservation

- 회의실
- 예약자 이름
- 예약자 이메일
- 사용 목적
- 시작 시간
- 종료 시간
- 예약 상태
- 생성 시간

데이터 구조는 최대한 단순하게 유지해서 설명 가능성과 구현 속도를 모두 챙겼습니다.

## 설계하면서 고민했던 내용

### 1. 인증 범위를 어디까지 둘지

처음에는 사용자/관리자 모두 계정을 두는 방식도 고려할 수 있었지만, 과제 특성상 짧은 시간 안에 회의실 예약이라는 핵심 기능을 완성하는 것이 더 중요하다고 판단했습니다.

그래서 사용자는 이메일 기반 조회로 단순화하고, 관리자는 세션 기반 비밀번호 로그인만 적용했습니다.

### 2. 프론트엔드 분리 여부

React 같은 별도 프론트엔드를 구성하면 화면 자유도는 높아지지만, Docker 구성과 개발 시간이 늘어날 수 있습니다.

이번 과제에서는 백엔드 로직과 요구사항 구현을 명확하게 보여주는 것이 더 중요하다고 보고 Thymeleaf를 선택했습니다.

### 3. DB 초기화 방식

회의실 기본 데이터를 바로 확인할 수 있도록 `data.sql`로 초기 데이터를 넣었습니다.

또한 재실행 시 중복 데이터가 쌓이지 않도록 `ON CONFLICT DO NOTHING` 형태로 처리했습니다.
